# Sprint 05 - 2026-03-11

## Objetivo da Sprint
Implementar a primeira versão pragmática de roteamento e ETA:
- Integrar provider externo (Mapbox ou Google) para cálculo de rota e ETA.
- Frontend desenhar polyline da rota retornada.
- Backend persistir distância/tempo para auditoria e precificação.
- Preparar caminho de evolução para OSRM/Valhalla no futuro.

## Itens Implementados
- Abertura da Sprint 05 e definição de escopo técnico inicial.
- Planejamento de arquitetura orientado a provider plugável para rota/ETA.
- Implementação de roteamento plugável no backend (`apps/trips/routing.py`) com provider inicial Mapbox.
- Integração do cálculo de rota no `TripRequest`:
  - Persistência de `estimated_distance_km`
  - Persistência de `estimated_duration_minutes`
  - Persistência de `route_polyline`
  - Persistência de `routing_provider`
- Inclusão de coordenadas de destino no domínio (`dropoff_latitude`/`dropoff_longitude`).
- Exposição dos dados de rota/ETA no serializer de resposta da corrida.
- Variáveis de ambiente adicionadas para roteamento:
  - `ROUTING_PROVIDER`
  - `MAPBOX_ACCESS_TOKEN`
  - `MAPBOX_PROFILE`
- Migração criada para novos campos de auditoria de rota na tabela `trips_trip`.
- Testes de API adicionados para:
  - persistência de rota quando provider retorna sucesso
  - fallback sem provider sem quebrar fluxo de solicitação de corrida
- Correções de suporte realizadas no ciclo:
  - Bruno salva tokens automaticamente no login (`passengerAccessToken`, `driverAccessToken`).
  - Bruno salva `tripId` automaticamente ao solicitar corrida.
  - Correção de variáveis de ambiente Bruno para JSON válido.
  - Patch de compatibilidade no Django Admin para ambiente local com Python 3.14.
